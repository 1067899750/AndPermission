Share private files

In order to improve the security of private files, the private directory for Android 7.0 or higher is restricted to access. Trying to pass file:// will trigger FileUriExposedException. The recommended way to share the content of a private file is using the FileProvider, it will generate a content:// URI.

The FileProvider class is located under the support v4 package, so it needs to dependency com.android.support:support-v4:version. AndPermission does not dependency Google's support package and compat package, so in order to avoid the increase in size, AndPermission copied FileProvider.

AndPermission provides a method for sharing files Uri compatible with Android 7.0 and higher:

String filePath = ...;

Uri uri = AndPermission.getFileUri(context, new File(filePath));

For example, when installing Apk on Android 7.0 or higher:

String apkPath = ...;

Intent intent = new Intent(Intent.ACTION_INSTALL_PACKAGE);
intent.setFlags(Intent.FLAG_ACTIVITY_NEW_TASK);
intent.addFlags(Intent.FLAG_GRANT_READ_URI_PERMISSION);
Uri uri = AndPermission.getFileUri(context, new File(apkPath));
intent.setDataAndType(uri, "application/vnd.android.package-archive");
context.startActivity(intent);

AndPermission provides an easier way to help developers improve the experience, please refer to Installing Apk on Android 7.0 and Android 8.0.
