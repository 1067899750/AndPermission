特别说明

特别说明

对于中国开发者而言，运行时权限是我们的一个痛点，AndPermission2.0发布之前邀请了很多中国开发者参与测试，测试并兼容了大量的中国产手机。虽然如此，但是在很多手机上测试结果还是不尽如人意，但是这些手机只是很少的一部分了，AndPermission也提供了合适的方法来兼容这种情况。

国产手机的各种情况比较多，这里就不一一例举说明了，这里说明一些使用AndPermission时出现的疑问。

为了兼容一些特殊中国产手机，AndPermission请求权限的流程做了一些优化：

                        发起请求
                            │
            ┌───────────────┴───────────────┐
            │                               │
    使用系统Api判断有权限            使用系统Api判断没权限
            │                               │
            │                               │
            │                        使用系统Api请求用户授权
            │                               │
            │                               │
            │                            用户授权
            │                               │
            └───────────────┬───────────────┘
                            │
                执行有权限的代码，是否发生异常
                            │
                        回调结果

虽然如此，但是在部分手机上，还是有一些权限不能真正的兼容到，比如发送短信、打电话、接听/挂断电话、定位、身体传感器、获取手机账户*等等。而这些权限往往都是绝大多数开发者用不到的（特殊开发者除外，下面会给出解决方案）。

比如身体传感器权限，是供手表使用的，在很多手机上没有身体传感器这个权限的管理开关。这个权限一般用来获取心率等，在目前市场上流行的手机上还测试到可以获取到这个传感器的手机，所以开发者也不必关注，如果你是手表开发者，那么这个权限不存在问题。

比如定位权限，当用户拒绝授权时，AndPermission在检查结果时内部实际上是执行了定位代码的，但是在很小一部分手机上，没有权限的时候并不会报错，用系统Api检查结果也是有权限的，这种情况实在是没办法了。开发者也可以直接调用定位代码，但是你可能拿不到定位信息或者直接定位失败。

比如电话权限，很多开发者申请时直接申请了电话权限组，但是有的手机中PROCESS_OUTGOING_CALLS权限是获取不到的，因为他只允许系统App使用这个权限，所以虽然用户同意了授权，我们接受到的结果还是失败，因此对于电话权限，使用什么权限就申请什么权限，不要申请权限组。

还有些权限也有同样的问题，这里不再过多解释，有问题的开发者可以在AndPermission/issues提问。

总结

有一个原则是使用什么权限就申请什么权限，不要过多的申请权限。比如电话、短信和联系人这种隐私级别比较高的要特别注意；比如日历、相机、定位、麦克风和存储空间这些常用且隐私级别不高的权限可以请求权限组，这些权限经过测试存在的问题也比较少。

AndPermission从2.0版本开始不再需要开发者从代码的实现上特别写什么判断了，按照文档中的规范使用即可。

另外，值得一提的是：为了防止有些手机没有权限时意外的返回成功，而真正执行代码时却发生崩溃，AndPermission在成功的回调方法上加了try catch，内部的实现如下：

private void callbackSucceed() {
    if (mGranted != null) {
        List<String> permissionList = asList(mPermissions);
        try {
            mGranted.onAction(permissionList);
        } catch (Exception e) {
            if (mDenied != null) {
                mDenied.onAction(permissionList);
            }
        }
    }
}

如上所以，当开发者执行有权限的代码发生异常时，AndPermission会抓到异常并回调到失败中，这里要注意的是会抓到任何异常，不仅仅是没有权限时的异常。
