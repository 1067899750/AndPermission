安装Apk

在Android7.0和Android8.0上安装Apk

Android从6.0开始发生了巨大的变化，例如Android6.0的运行时权限，Android7.0的私有文件保护原则，Android8.0的未知来源应用安装权限等，以上三点对应用内安装Apk都有较大的影响。

不使用AndPermission的做法

如果App的targetSdkVersion大于等于23，当App运行在Android6.0的设备上时，安装Apk时App需要申请到外部设备读写权限（如果Apk文件不是保存在SD卡则不用），此时开发者通过以下代码即可正常启动安装：

String apkPath = ...;

Intent intent = new Intent(Intent.ACTION_INSTALL_PACKAGE);
intent.setFlags(Intent.FLAG_ACTIVITY_NEW_TASK);
Uri uri = Uri.fromFile(new File(apkPath));
intent.setDataAndType(uri, "application/vnd.android.package-archive");
content.startActivity(intent);

当App运行在Android7.0的设备上时，安装Apk时仅拥有外部存储设备读写权限是不够的。因为Android7.0的私有文件保护原则，所以尝试传递file://URI 会触发FileUriExposedException，Android开发者网站推荐我们使用FileProvider来共享文件，但是为了避免引入support v4包造成的宝体积增大问题，AndPermisison拷贝了FileProvider类并提供了兼容Android7.0及更高系统中分享私有文件的方法，详情请参考分享私有文件。

当App运行在Android8.0的设备上时，安装Apk时需要在manifest.xml文件添加REQUEST_INSTALL_PACKAGES权限，开发者可以通过PackageManager#canRequestPackageInstalls()判断用户是否为App打开了未知来源安装开关，开发者可以通过以下代码跳转到设置页面：

Intent intent = new Intent(Settings.ACTION_MANAGE_UNKNOWN_APP_SOURCES);
intent.setData(Uri.fromParts("package", getPackageName(), null));
startActivity(intent);

抛开运行时全线不说，仅仅Android7.0和Android8.0的这些操作写起来就有些繁琐了。

使用AndPermission的做法

特别注意：使用AndPermission也需要在manifest.xml中添加REQUEST_INSTALL_PACKAGES权限，并且调用AndPermission的安装代码之前需要保证App拥有外部存储设备读写权限。

最简单的时候只需要传入apk文件路径即可完成整个安装流程：

String apkPath = ...;

AndPermission.with(this)
    .install()
    .file(new File(apkPath))
    .start();

此时如果没有REQUEST_INSTALL_PACKAGES权限，AndPermission将会自动启动设置页面让用户授权，用户授权完成回来后，AndPermission会根据是否有权限继续安装或者退出安装。

如果开发者想把体验做的更好一点，例如没有权限时弹一个Dialog让用户选择是否启动设置，用户同意或者拒绝后做一个统计：

String apkPath = ...;

AndPermission.with(this)
    .install()
    .file(new File(apkPath))
    .rationale(new Rationale<File>() {
        @Override
        public void showRationale(Context c, File f, RequestExecutor e) {
        }
    })
    .onGranted(new Action<File>() {
        @Override
        public void onAction(File data) {
        }
    })
    .onDenied(new Action<File>() {
        @Override
        public void onAction(File data) {
        }
    })
    .start();

当rationale()被回调后说明没有REQUEST_INSTALL_PACKAGES权限，此时开发者必须回调RequestExecutor#execute()来启动设置或者RequestExecutor#cancel()来取消启动设置，否则将不会回调onGranted()或者onDenied()中的任何一个，也就是说AndPermission将不会有任何响应。

当rationale()被回调时正确的做法是显示一个Dialog征求用户意见，是否要启动设置页面授权未知来源安装权限，根据用户选择调用RequestExecutor#execute()或者RequestExecutor#cancel()。

当onGranted()被回调时说明用户同意了权限，并会立刻进入安装界面，用户点击安装系统将会自动安装目标Apk到系统中；当onDenied()被回调时说明用户拒绝了权限，并会立刻退出安装界面。开发者可以在这两个方法被调用时做一些必要的统计。
